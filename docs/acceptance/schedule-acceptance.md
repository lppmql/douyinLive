# 主播排班验收

## 前置条件

- MySQL 已导入当前生效的真实排班模板。
- 查询日期范围内存在真实直播场次。

## 操作步骤

1. 查询单日、近 7 天和自定义起止日期。
2. 核对文豪、大全每天 4 场，其余当前排班主播每天 3 场。
3. 抽查不足 45 分钟、45 至 79 分钟、达到 80 分钟和额外加场的真实场次。
4. 核对实际开播是否落在计划时间所在自然小时，跨出时显示跨整点提醒。
5. 点击缺场、无效、时长不足或加场提醒，穿透到对应日期和真实场次。

## 预期结果

- 同一真实场次只匹配一个计划班次。
- 不足 45 分钟的已结束场次标记无效；直播中场次不提前判无效。
- 超出规定场次显示加场，并在主播完成度中按日期汇总。
- 未来班次不提前产生缺场提醒。

## 失败与恢复

- 主播未匹配：检查主播别名与排班模板，不使用昵称猜测替代真实映射。
- 日期数量异常：核对查询范围包含首尾两天且不超过 31 天。
- 规则调整：先更新模板或规则来源，再重新运行匹配，禁止直接改历史真实场次。
