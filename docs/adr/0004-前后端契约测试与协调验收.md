# ADR 0004: 前后端契约测试与协调验收

- **日期**: 2026-07-18（修复）/ 2026-07-20（类型安全强化）
- **状态**: accepted
- **决策者**: 项目维护者

## 背景

2026-07-18 前后端协调验收发现 **8 个已修复问题**：

1. API 参数命名不一致：`taskId`（前端 camelCase） vs `task_id`（后端 snake_case）
2. 前端残留 SoybeanAdmin 模板接口（`/route/getConstantRoutes` 等指向 mock 地址）
3. 前端请求层残留 `ApifoxToken`（SoybeanAdmin 模板的 mock 鉴权）
4. WebSocket 默认地址指向 `mock.apifox.cn`
5. Redis 重复订阅导致广播消息重复
6. 采集日志接口参数不一致
7. 前端调用后端不存在的接口
8. 话术分段响应字段不一致

## 决策

### 2026-07-18：运行时契约测试

新增 `test_frontend_backend_contract.py`：
- 从 FastAPI OpenAPI schema (`app.openapi()`) 提取后端所有端点
- 从前端 `src/service/api/*.ts` 源码中解析 `backendRequest()` 调用
- 运行时对比——前端调用的每个 URL 必须在后端 OpenAPI schema 中存在
- 额外检查：参数命名一致性（`task_id` 而非 `taskId`）、无残留 Apifox Token、WebSocket 使用代理

### 2026-07-20：类型安全强化

- 后端 ~40 个裸 dict 端点补齐 Pydantic `response_model`（新增 `schemas/ai.py`、`transcript.py`、`dashboard.py`、`knowledge.py`）
- 安装 `openapi-typescript`：`pnpm gen-api` 从 `/openapi.json` 自动生成前端类型
- 目标：后端改 Pydantic 字段 → CI 检测前端类型不同步 → 构建变红

## 后果

- ✅ 编译期 + 运行时双重契约保护
- ✅ 不再出现"前端调用了不存在的接口"这类 bug
- ⚠️ `openapi-typescript` 生成的类型尚未全面替换手写 `douyin.d.ts`（渐进迁移中）
- ⚠️ 内部服务（collector/ASR/AI）之间仍通过裸 dict 通信，未引入 Pydantic 类型校验
