# ADR 0007: README 重构与文档职责划分

## 状态

已采纳（2026-07-21）

## 背景

原 README.md 共 433 行，混合了项目简介、功能详解、开发记录、运维手册、配置细节等多种内容。对新手来说信息密度过高、难以快速找到重点；对维护者来说，同一份内容在 README 和 docs/ 中重复维护，容易不一致。

## 决策

### README.md 的职责（~120 行）

README 是**项目门面**，只回答三个问题：

1. 这个项目是什么？（一句话 + 技术栈）
2. 怎么跑起来？（快速开始 4 步）
3. 去哪找更多信息？（文档导航表格）

### docs/ 的职责

| 文档 | 覆盖内容 |
|------|---------|
| `docs/开发.md` | 环境搭建、目录结构、开发流程、代码规范、职责分层 |
| `docs/部署.md` | 首次部署、发布流程、回滚、备份、安全检查 |
| `docs/故障排查.md` | 后端/前端/数据库/Docker/性能 常见问题按症状→诊断→解决 |
| `docs/adr/` | 架构决策记录（为什么这样设计），只增不改 |
| `docs/beginner-guide.md` | 图文操作教程 |
| `CHANGELOG.md` | 版本变更记录（按日期倒序） |

### 具体变化

从 README 移除的详细内容，全部已有对应文档覆盖：

| 原 README 章节 | 去向 |
|---------------|------|
| 核心能力（26 条详细描述） | 精简为 10 条要点 + 链接到开发.md |
| DataEase 大屏详细配置 | 部署.md 已有 |
| 推荐操作顺序 | beginner-guide.md 已有 |
| 主播排班详细规则 | 功能页面 + 开发.md |
| 场次详情与 AI 复盘详解 | 开发.md |
| 前端交互规范 + 日期记录 | CHANGELOG 已有，规范在开发.md |
| ASR 安全说明（详细） | 故障排查.md + 开发.md |
| 知识库内容（详细） | 开发.md |
| 可观测性（详细） | 部署.md |
| 数据安全（详细） | 保留要点，详情在部署.md |

## 后果

- **正面**：README 对新人和路人都友好；维护者只需在一处更新内容
- **负面**：需要确保 docs/ 下的文档保持更新（已有 make doctor 检查机制）
- **注意**：doc 文件名已改为中文（2026-07-21），引用路径需同步更新
