# ADR 0005: 前端 UI/UX 优化与环境变量整合

- **日期**: 2026-07-20
- **状态**: accepted
- **决策者**: 项目维护者

## 背景

项目前端基于 SoybeanAdmin 框架，但存在以下问题：
- 图标按钮无 aria-label，屏幕阅读器无法使用
- 触控目标 36px，低于 Apple HIG 44px 标准
- 间距不统一（6px/7px/9px/18px 混用），非 8px 节奏
- 图标风格混用（gridicons/line-md/ph/heroicons + mdi）
- 颜色硬编码无法适配暗色模式
- 环境变量分散在 6 个 .env 文件（根目录 + frontend/ 各有多个）

## 决策

### UI/UX 优化

1. **无障碍**：`button-icon.vue` 自动从 `tooltipContent` 派生 `aria-label`；触控目标 36→44px；`business-focus-ring`/`business-active-press` 通用类
2. **间距规范**：全站 gap/margin/padding 统一为 8px 倍数
3. **暗色模式**：硬编码颜色→CSS 语义变量（`rgb(var(--primary-color))`）；对比度 0.035→0.06
4. **图标统一**：全部改为 `mdi:`（SoybeanAdmin 默认图标集）
5. **安全区域**：`safe-area-inset-*` 适配刘海屏/灵动岛
6. **响应式**：断点 768px→767px 对齐 UnoCSS

### 环境变量整合

1. 前后端 `.env` 分离：根 `.env` 纯后端变量（8 板块）、`frontend/.env` 纯 VITE_* 变量
2. `.gitignore`：`.env` → `/.env`，只忽略根目录含密钥的 .env
3. 删除 `packages/alova/`（项目零引用，实际用 `@sa/axios`）
4. 删除 `.env.prod`/`.env.test`（SoybeanAdmin 模板 mock 地址）
5. 新增 `CORS_ORIGINS` 统一变量，`main.py` + `docker-compose.yml` + `dataease/conf/application.yml` 三处统一引用

## 后果

- ✅ 全站 UI 一致性：触控 44px、间距 8px 节奏、颜色语义变量
- ✅ 环境变量从 6 个文件收敛到 2 个（根 + frontend/）
- ✅ CORS 配置只需改一行（`CORS_ORIGINS`），三处自动同步
- ⚠️ 前端 831 行手写类型（`douyin.d.ts`）尚未完全替换为自动生成
