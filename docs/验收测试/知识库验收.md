# 知识库验收

## 前置条件

- 至少有两位真实主播完成话术、评论、分钟指标和复盘同步。
- DeepSeek 配置可用。

## 操作步骤

1. 等待后台同步全部待处理场次，记录新增、更新、跳过和失败数量。
2. 提问一个指定主播、指定场次或指定日期的问题。
3. 展开引用来源，核对场次 ID、时间片和原始评论或话术。
4. 连续追问后切换到另一位主播，确认新问题的明确范围优先于旧上下文。
5. 点击引用跳转直播场次详情，确认时间和内容一致。

## 预期结果

- 回答只基于真实知识切片，并展示可追溯来源。
- 同步重复执行保持幂等，不重复插入相同场次时间片。
- 明确指定的新主播、新场次和时间范围覆盖上一轮继承上下文。
- 数据不足时直接说明缺失，不生成虚构经营结论。

## 失败与恢复

- AI 超时：保留当前对话与检索来源，允许重试。
- 来源为空：先检查场次是否完成 ASR 和知识同步。
- 上下文串主播：清空当前范围后重新指定主播，并保留问题用于回归测试。
