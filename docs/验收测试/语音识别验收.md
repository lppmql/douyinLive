# ASR 话术验收

## 前置条件

- 选择一场真实、已结束、回放有声音且时长不少于 45 分钟的场次。
- FunASR 运行正常，场次保存了可用 m3u8 地址。

## 操作步骤

1. 为真实场次创建转写任务，记录任务 ID、场次 ID 和 Trace ID。
2. 至少完成两个分片后停止 ASR Worker，再重新启动。
3. 验证已完成分片保留，未完成分片继续执行。
4. 转写完成后核对分片起止时间、全文、覆盖时长和最新任务状态。
5. 使用同一场次再次点击重试，确认不会删除成功分片。

## 预期结果

- 同时只运行一个重任务，最新真实场次优先，人工高优先级任务可以插队。
- Worker 重启后只续跑未完成或失败分片。
- 全文写入 `LONGTEXT`，长场次不会因普通 `TEXT` 容量失败。
- 无音频、回放过期和引擎不可用均显示具体原因与恢复动作，不写入模拟话术。

## 失败与恢复

- m3u8 过期：先刷新场次回放地址，再继续转写。
- FunASR 不可用：在采集页重启 ASR，确认健康后重试。
- 数据库断开：恢复 MySQL 后重试，已完成分片不得重复识别。
